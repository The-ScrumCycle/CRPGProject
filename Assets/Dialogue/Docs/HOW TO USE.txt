LineNode:
- two types: Npc, Player
    - Npc:
        - will be processed by DialogueRunner and invokes the action sent to the ui
        - this is what will be used for the NPC dialogue

    - Player:
        - usually is pointed from an options node. DialogueRunner will skip this node as it will not need to be processed
        to view player options in UI.

- contains the actual text that will be displayed in the UI.
- action contains a keyword that the Controller/Runners will listen for to perform. 
- instead of choose the index in the array, more clear to look for a keyword associated with a node
- JSON structure:

        {
            "id": "4",
            "type": "line",
            "speaker": "player",
            "text": "I'd like to get on the boat.",
            "action" : "board",
            "minIntelligence": 0,
            "next": "6"
        },

OptionalNodes:
- used by a player, contains an array of LineNodes/ConditionalNodes that the player can chose from
- DialogueRunner will parse through options, check each lineNode minIntelligence level and then will display options based on Events/minIntelligence
will invoke an action to ui containing an array of strings meeting the requirements
- must have the following attributes in a JSON file

{
    "id": "2",
    "type": "option",
    "options": ["3", "4", "5"]
}

- options represents the array of node ids the user can chose from

ConditionalNodes:
- contains two nodes, a pointer to a node if the condition is met, a pointer to a node if condition is not met.
- allows player/npc to say different things based on events that have happened in the game.
- JSON structure:

        {
            "id": "4",
            "type": "conditional",
            "eventName" : "beat_boss",
            "trueNode" : "2",
            "falseNode" : "1",
        },



DIALOGUE WORKFLOW:
1. Create JSON file containing all the dialogue, refer to Assets/Dialogue/Data/NPC/captain.json for an example.
2. make sure your npc controller has the following attribute --> [SerializeField] private UIRunner uiRunner;
3. Go to your npc controller.cs and add the following to Awake()

    private void Awake()
    {
        state = GameState.Instance;
        // find the ui runner in scene if not assigned in inspector
        if (uiRunner == null)
        {
            uiRunner = FindObjectOfType<UIRunner>();
        }

        if (uiRunner == null)
        {
            Debug.LogError("CaptainController: No UIRunner found in scene.");
            return;
        }
        // load captain dialogue graph and create a runner component
        uiRunner.InitializeDialogue(from:gameObject, characterName:"captain");
        uiRunner.OptionSelectedAction += OnOptionSelected;
        uiRunner.DialogueEndedAction += OnDialogueEnded;
    }

you will need to perform overrides for OnOptionSelected, OnDialogueEnded
OnOptionSelected function signature is --> OnOptionSelected(string action) --> where action is a user action to perform, ex: "board ship"
OnDialogueEnded --> what do you want the player to do once dialogue ends

Add the following to make sure to unsubscribe from uiRunner actions.

    private void OnDestroy()
    {
        // unsubscribe to avoid duplicate callbacks after domain reloads
        if (uiRunner == null)
        {
            return;
        }

        uiRunner.OptionSelectedAction -= OnOptionSelected;
        uiRunner.DialogueEndedAction -= OnDialogueEnded;
    }

In the function OnMouseDown() or Interact() --> and a BeginNpcDialogue() call and implement the function yourself
**for simplicity, check if uiRunner is not null and call uiRunner.BeginDialogue(string npcname)



GOOD TO KNOW:
- dialogue structure for player: dialogue for the player must always live in an OptionalNode, if there is no OptionalNode for a player, it's lineNode will
not be displayed on the UI
- npc dialogue always lives in a lineNode
